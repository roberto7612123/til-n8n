### Context
n8n을 활용하여 Discord에서 메시지를 수신할 때 자동화된 워크플로우를 만들었습니다. 이 워크플로우는 메시지 수신을 트리거로 사용하여 TIL 형식으로 변환한 후 GitHub에 코드를 푸쉬(pull)하고, README.md 파일에 인덱스를 추가하고, 마지막으로 완료 메시지를 Discord에 발신합니다.

### Core
```json
{
  "nodes": [
    {
      "name": "Discord Trigger",
      "parameters": {},
    },
    {
      "name": "Agent Execution",
      "parameters": {
        "conversion": "TIL"
      },
    },
    {
      "name": "GitHub Push",
      "parameters": {
        "repository": "your-repo",
        "filePath": "path/to/TIL"
      },
    },
    {
      "name": "Update README",
      "parameters": {
        "filePath": "README.md"
      },
    },
    {
      "name": "Discord Message",
      "parameters": {
        "content": "Workflow completed"
      },
    }
  ]
}
```

### Insight
n8n을 처음 사용해 보았지만, 노드의 입력(Input)과 출력(Output)을 이해하면 워크플로우 구축이 매우 쉬웠습니다. 복잡해 보이는 자동화도 몇 가지 핵심만 알면 쉽게 설정할 수 있었습니다. 이 모든 것을 알게 해준 후츠릿에게 감사를 전합니다.
